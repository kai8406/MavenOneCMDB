The minimalistic Asset Management application

The Asset Management Application displays basic information about CI:s (Configuration Items) in the CMDB. Data is presented in tables with a row for each CI and a column for each attribute (field). It is currently neither possible to sort data, nor add or remove columns.

The tree view in the leftmost pane organises all CI:s in the same way as they are defined in OneCMDB. All instances (CI:s) in OneCMDB are based on Templates, which are structured in an object-oriented (class-oriented) fashion. This means that a root-CI-template exists that contains a few, generic attributes. Templates beneath the root-CI-template inherit all attributes from the root-CI-template. Each template in the tree inherits from the template above, its parent. For each level down the templates become more and more specialized and rich.

The number within parenthesis behind each template name in the tree view states how many instances (CI) that exists on this level. I.e. based on this level´s template, or any other template (child) further down in the tree.
